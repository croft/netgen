Abstract specification corresponding to the SyncTask task class in Scala. Tasks that extend SimpleTask should make their implementation extend the SimpleTask class.

A SimpleTask has a simpler state machine than BaseTask. It should be used only for tasks that can be repeated without issue (e.g. pure functions like StringReplace, and simple tasks that are OK to be executed multiple times like Log). It cannot be cancelled, so it should only be used for tasks that do not involve holding resources and that are quick to execute.

Note that start and end nodes are in capital letters.

task SyncTask(task_implementation_class, driver_meta?)

Abstract specification corresponding to the BaseTask class in Scala. Tasks that extend BaseTask should make their implementation extend the BaseTask class.

This is a more heavy weight task than SimpleTask. It requires more transactions to execute, because its state machine is more complex. On the other hand, it offers more guarantees, like ensuring that a task is not repeated even if the driver crashes, and providing cancellation means.

Note that BaseTask here does not extend the SimpleTask task. That’s because it makes it clearer to have the whole specification of the state machine in a single task definition.

task AsyncTask(task_implementation_class, driver_meta?)
